## 1. 各家平台差异化分析

### 1.1 携程（Ctrip / Trip.com Group）——「全品类 + 高端服务 + 全球供应链」龙头

[查证 + 推断] 携程占据中国在线旅游市场约 71% 份额（维基百科 2024 数据），是票券类目**完整度最高**的玩家。其差异化体现在三个层次：(a) **品类最全**——把"票券"做成一个完整产品族：景区门票、演出票（演唱会/音乐节/话剧/赛事）、当地玩乐（一日游/体验课/手作）、邮轮、滑雪、潜水、SPA、酒店套餐，分类树最深（三级以上：演出 → 演唱会 → 按艺人/场馆二级）；(b) **服务链条最长**——"超级会员"提供电子导览、机场贵宾厅、行李寄送、24h 客服等票券周边的**履约增强服务**，把"卖票"延展为"出行管家"；(c) **价格保障承诺**——首页直接写"价格透明，退票改签不收取额外费用"+"确认订单后不涨价"（查证），用承诺型文案降低用户决策摩擦。其票券系统**最显性的工程化壁垒是"按日库存 + 二次确认"双模态**——热门景区（迪士尼/环球影城/故宫）走"按日库存 + 实名分时段"（instant confirmation），小众景区/民宿体验走"on-request + 24h 确认"。

### 1.2 同程旅行（Tongcheng Travel）——「下沉 + 火车票联动 + 履约保障」下沉市场之王

[查证 + 推断] 同程旅行的差异化是**对下沉市场与履约保障的极致优化**。其票券业务有三条独特主线：(a) **黑鲸会员**体系（查证首页出现的"一元特权卡 / 专属旅游顾问 / 同程驿站特权 / 投诉优先处理"等权益为佐证），主打"48 小时无忧退 / 一键全退"（业内公开资料多次提及），将**退改保障**作为会员核心价值锚，把"票券"从一次性商品变成可信赖的服务承诺；(b) **火车票联动**——同程在火车票市场份额第二（仅次于 12306），其票券系统天然支持"火车票 + 门票"+"火车票 + 酒店"+"机票 + 酒店 + 门票"组合套餐，SKU 组合爆炸式增长；(c) **驿站 / 城市卡**——"同程驿站"是同程独有的线下履约点（机场/高铁站柜台），用户到驿站可现场改签、补打凭证、跨城市取票，这一"线下柜台"网络是其他两家没有的护城河。

### 1.3 飞猪（Fliggy）——「阿里生态 + 信用 + 会员通」流量与金融化先锋

[查证 + 推断] 飞猪的票券业务核心是**阿里生态的横向打通**。其差异化集中在三处：(a) **支付与信用**——背靠支付宝，是业内最早做"信用住 / 先住后付 / 免押"的玩家（"信用住"在 iyiou 关键词库中作为飞猪代表词出现，查证），把芝麻信用分嵌入票务全流程（先玩后付 / 免押租赁 / 信用游）；(b) **88VIP 会员通**——阿里 88VIP 是横跨淘宝/天猫/优酷/饿了么/飞猪的联合会员，飞猪票券是其权益池的一部分（"88VIP"作为飞猪代表词在公开资料中反复出现，查证），用户买 88VIP 后在飞猪享专属价、专享库存、退款加速；(c) **飞猪里程 + 内容化**——"飞猪里程"作为平台积分（查证），叠加"直播带货 + 达人探店"等内容化形态，把票券和内容/直播/达人分销深度绑定。**飞猪最弱的是线下履约**（没有自营驿站），但**最强的是支付与流量**——阿里系 10 亿+ 月活是飞猪票券的天然漏斗顶。

## 3. 差异化竞争力（独特功能 — 10+ 项）

1. **携程 · 超级会员价 + 价格保障承诺**：会员价 + "确认订单后不涨价"白纸黑字写在首页（查证）——这是把"履约确定性"做成产品特性。
2. **携程 · 电子导览**：购票后内置"扫码即听"语音导览（公开资料推断），把票券延展为内容产品。
3. **携程 · 机场贵宾厅 + 行李寄送**：票券订单可加购"高铁站/机场贵宾厅休息 + 行李同城寄送"——出行前/中/后的服务闭环。
4. **携程 · 24h 客服 + 一键海外救援**：在跨境票券（出境游/海外景区）上是事实标准。
5. **同程 · 黑鲸会员 48 小时无忧退**：业内最具传播力的退改承诺（推断 + 公开资料反复提及），把"票券可退"做成品类心智。
6. **同程 · 火车票联动套餐**：火车票 + 门票 + 酒店的三段式套餐（推断 + 同程火车票市场份额第二），用"一次购齐出行链"提高客单。
7. **同程 · 同程驿站**：线下柜台网络（机场/高铁站），提供现场改签、补打、跨城取票——纯线上玩家无法复制的护城河。
8. **同程 · 签到游币 + 红包雨**：把"低频高客单"的票券，做成"日活可参与"的游戏化形态（查证首页权益）。
9. **飞猪 · 信用住 / 先玩后付**：背靠支付宝芝麻信用，免押租赁、免押入住、先玩后付——金融化票券的代表（查证关键词存在）。
10. **飞猪 · 88VIP 跨平台会员通**：88VIP 一份钱通享淘宝/天猫/优酷/饿了么/飞猪权益（查证 + 公开资料）——把票券放进"生活方式订阅"。
11. **飞猪 · 飞猪里程 + 达人直播**：积分体系 + 直播带货 + 达人探店，把票券和内容深度绑定（查证关键词存在）。
12. **飞猪 · 阿里生态联动**：高德地图导航、菜鸟行李、饿了么订餐——把"出游"打成"在阿里生态里走一圈"的体验。
13. **[三家共有] AI 行程规划**：携程"AI 行程助手"、同程"AI 旅行助手"、飞猪"AI 游伴"——把票券作为"AI 行程"中的可执行节点。
14. **[三家共有] 动态码 + 防截图**：凭证二维码定时刷新，防截屏盗用（推断为通用方案）。

## 4. 对 TicketHub 单类目系统的可移植洞察（按 ROI 排序）

> ROI 估算 = (用户价值 × 实现成本倒数) × 平台契合度。**优先做 P0（高 ROI、低成本、与单类目契合），慎重做 P3（成本高、跨类目才值得）。**

### P0 —— 必须做（高 ROI，能直接抄）

1. **价格分人群 + 平日/周末/节假日差价**【成本：低 / 价值：高】
   - TicketHub 已有 `Product.attributes` 的 Mixed 字段，**只要在策略层（`src/lib/strategies/sight.ts`）加 `getPriceFor(visitorType, visitDate)`**，就解决了演出/景区/玩乐**最刚需**的"同票不同价"问题。
   - 这是演出票（成人/儿童/学生/老人）和景区（家庭套票）的**事实标准**，不做就上不了规模。

2. **按日库存 + 库存预占**【成本：中 / 价值：高】
   - Cycle 0 `EVOLUTION.md` 已识别此为缺口。`Product` 模型需要新增 `dailyInventory: Map<Date, {total, sold, held}>`，下单时事务性 `held++`，支付成功后 `sold++` 并 `held--`，超时/取消时 `held--`。
   - 没有按日库存，**任何热门景区票（迪士尼/故宫级别）都无法上线**。

3. **退款政策矩阵 + 阶梯式手续费**【成本：低 / 价值：高】
   - `Order` 加 `refundPolicy: { tiers: [{ hoursBeforeVisit, feeRate }] }`，退款时按出行时间差匹配档位。
   - 演出票/景区票的退款是最敏感决策点，**不透明就直接流失订单**。

4. **凭证改期 + 换人**【成本：低 / 价值：高】
   - 演出票/景区票"买了去不了"是高频场景，目前 Voucher 模型需要扩展 `transfersRemaining` + `nameChangeDeadline`。
   - 不支持改期 = 转化率直接打折。

### P1 —— 应当做（中 ROI，单类目内必需）

5. **有效期模型分三态**【成本：中 / 价值：中】
   - 当前 Voucher 的 `expiresAt: Date` 是单点。要支持：`fixedDate`（指定日）/ `validForDays`（N 天内有效）/ `validUntil`（有效期至某日）三态。
   - 演出票是 `fixedDate`、景区年卡是 `validUntil`、当地玩乐是 `validForDays`——三态都常见。

6. **价格日历**（详情页展示未来 30/60 天的价格） 【成本：中 / 价值：中】
   - 直接驱动"低价日历"这个流量入口（携程/飞猪详情页都有）。
   - 底层依赖 #2 按日库存 + #1 价格分人群。

7. **即时确认 vs 二次确认 双模态**【成本：低 / 价值：中】
   - `Product.confirmationMode: 'instant' | 'on-request'`，on-request 时 Order 状态多一档 `pending-confirmation`，由商家/系统异步确认。
   - 当地玩乐小供应商几乎都是 on-request，不支持就别想做小众玩法。

### P2 —— 可选做（低 ROI 或需要业务规模支撑）

8. **多商家券叠加规则**【成本：高 / 价值：中】
   - "平台券 × 商家券 × 银行券"的叠加规则是营销中台——TicketHub 现在没有营销中台，**先上单层券即可**，三券叠加是 P2。

9. **AI 行程规划**【成本：极高 / 价值：中】
   - 单类目系统不必自建，**接 LLM API 即可**（参考 AI-SPEC 阶段的设计）。但 ROI 取决于用户是否真的"用 TicketHub 规划一整天的行程"——这其实是流量问题。

10. **电子发票 + 自助开**【成本：中 / 价值：低】
    - 商旅/企业客才高频需要，C 端用户大多不要。当前阶段**对接第三方电子发票服务**（如票易通）即可，0 自研。

### P3 —— 不要做（跨类目、ROI 极低或与单类目哲学相悖）

- **酒店 + 机票 + 门票联动套餐**：TicketHub 故意只做单类目（CLAUDE.md §0），**不要做**。
- **线下驿站 / 柜台网络**：重资产，单类目 SaaS 撑不起。
- **信用住 / 先玩后付**：需要金融牌照 + 阿里/腾讯级别风控，**不要做**。
- **88VIP 跨平台会员通**：需要生态，不是 TicketHub 的赛道。

## 6. 结论与下一步

1. **TicketHub 在"商品模型"的"按日库存 + 价格分人群"上是最大缺口**——直接阻碍热门景区/演出票上线（P0-1, P0-2）。
2. **退款政策与凭证灵活性是转化率的关键卡点**——P0-3, P0-4 是 v0 上线前必须补齐的最小集。
3. **"超级会员 / 黑鲸 / 88VIP"差异化竞争力是"履约承诺 + 生态联动"**——TicketHub 在单类目哲学下**不应**做跨平台会员通，但**应**做"履约承诺"型的会员价（例：T 币会员 + 7 天无忧退 + 改期免手续费）。
4. **"票券 + AI 行程"是 2025-2026 共同演进方向**——已记录在 `AI-SPEC.md` 路线中，无需新启动。
5. **P3 的功能（线下驿站 / 信用住 / 跨类目套餐）**与单类目哲学相悖，**应明确不做**，并在 `EVOLUTION.md` 中写明否决原因。
